# Cursor Context Hub

This folder centralizes agent operating rules and project context.

## Files

- `architecture.md`: Front-end architecture principles
- `design-system.md`: UI/UX standards and token logic
- `components.md`: Component registry and entry template
- `workflow.md`: Mandatory execution flow from analysis to delivery
- `tasks.md`: Active and completed task tracking
- `memory.md`: Persistent decisions and decision log

## Usage

1. Read relevant context files before implementation.
2. Do not start coding without analysis and task planning.
3. Document new decisions and components within the same session.
